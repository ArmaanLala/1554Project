Problem Description

The core problem can be described as how many numbers can be dialed when a user is only allowed to dial a keypad in an “L-shaped” pattern, similar to that of a knight in chess. Further, “Linear dialing” explores the possible combinations of numbers given this constraint, and investigates what kind of numbers of any given length can be generated. For instance, shown to the right, starting with 7, one can either take the red or the blue L-path to reach the next number. This means that any number following a 7 must either be a 2, or a 6. Further, any number following a 2 must be either a 9 or go back to 7, and so on. It is worth noting that with the given constraint pattern, the number 5 would never be utilized.

Background

The idea was given to our group by our studio instructor Santana Afton, and bridges the gap between computing and linear algebra in a way we can meaningfully investigate given the condensed summer session. Further, Linear Dialing has its roots in adjacency matrices and using those to find a solution set for any given number. Shown below is an L-path adjacency matrix for the valid numbers shown above.

Solution of Interest

Our solution is to use adjacency matrices and their related operations to calculate the various combinations one could achieve using Linear Dialing. By computing the adjacency matrices, we can then find unique solutions for each and every number that can be computed by Linear Dialing. Using this data, we can then appropriately use Eigenvalues, vectors, and other linear algebra concepts to further analyze the data.
